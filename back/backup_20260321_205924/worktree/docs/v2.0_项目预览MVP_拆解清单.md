# Smart Code Ark v2.0 项目预览 MVP 拆解清单

- 文档版本：v2.0
- 生成日期：2026-03-20
- 目标范围：仅在 `generate finished` 和 `modify finished` 两个节点触发预览

## 1. v2.0.0（基础可用）拆解清单

### 1.1 数据库与模型

1. 新增迁移脚本：创建 `task_preview` 表。
2. 增加唯一索引：`uk_task_id(task_id)`。
3. 增加查询索引：`idx_user_status(user_id, status)`。
4. 增加过期索引：`idx_expire_at(expire_at)`。
5. 新增后端实体：`TaskPreviewEntity`。
6. 新增仓储接口：`TaskPreviewRepository`。

### 1.2 后端核心能力

1. 新增 `PreviewDeployService`，负责预览实例部署、状态写回、失败回写。
2. 在 `AgentOrchestrator` 任务完成逻辑中增加触发判断：
3. 条件：`task.status == finished && task.taskType in (generate, modify)`。
4. 非目标任务类型直接跳过预览触发。
5. 触发方式采用异步，不阻塞任务主流程返回。
6. 新增 `TaskPreviewResult` 字段：`taskId/status/previewUrl/expireAt/lastError`。
7. 改造 `GET /api/task/{taskId}/preview`，返回真实预览状态。

### 1.3 前端页面能力

1. 改造结果页预览展示，接入真实状态字段。
2. `provisioning` 状态轮询查询（2-3 秒一次）。
3. `ready` 状态加载 iframe 并显示“新窗口打开”入口。
4. `failed` 状态显示失败信息占位。
5. `expired` 状态显示“预览已过期”提示。
6. 保持下载 ZIP 与修改生成功能不变。

### 1.4 可观测与配置

1. 新增预览部署日志：开始、成功、失败、耗时。
2. 新增开关配置：`preview.enabled`。
3. 新增开关配置：`preview.autoDeployOnFinish`。
4. 增加默认 TTL 配置：`preview.defaultTtlHours=24`。

### 1.5 联调与验收

1. 生成任务完成后，接口返回 `provisioning`。
2. 部署成功后状态变为 `ready` 且返回可访问 `previewUrl`。
3. 修改任务完成后重复上述流程，生成新的预览。
4. 非 `generate/modify` 类型任务不触发预览部署。
5. 结果页在 `ready` 前不出现空白死页面。

## 2. v2.0.1（可恢复）拆解清单

### 2.1 后端接口与能力

1. 新增接口：`POST /api/task/{taskId}/preview/rebuild`。
2. 仅允许 `failed` 或 `expired` 状态触发重建。
3. 重建成功后状态回到 `provisioning`。
4. 增加错误码映射：构建失败、启动失败、代理失败、超时失败。

### 2.2 前端交互

1. `failed/expired` 状态展示“重建预览”按钮。
2. 点击重建后按钮进入 loading，防重复提交。
3. 重建成功后自动进入轮询。

### 2.3 验收

1. 失败任务可重建并最终进入 `ready`。
2. 重建中页面状态可见，不出现误导性成功提示。

## 3. v2.0.2（资源治理）拆解清单

### 3.1 生命周期治理

1. 增加定时任务：扫描过期预览并回收运行实例。
2. 回收后将状态置为 `expired`。
3. 清理异常实例并写入错误日志。

### 3.2 配额限制

1. 增加“每用户并发预览数”限制，默认 2。
2. 超限时返回明确错误码与提示文案。

### 3.3 验收

1. 到期预览在目标窗口内完成回收。
2. 用户超限时前后端提示一致且可理解。

## 4. v2.0.3（体验增强）拆解清单

### 4.1 状态体验

1. `provisioning` 细分展示阶段文案。
2. 可选引入 SSE 推送，失败时自动回退轮询。

### 4.2 验收

1. 长时间部署场景下用户可持续获得状态反馈。
2. SSE 不可用时功能不回归。

## 5. 角色分工建议

1. 后端：数据表、服务编排、接口、定时回收、日志与配置。
2. 前端：结果页状态机、轮询/SSE、重建交互、错误态展示。
3. 测试：功能回归、状态流转、异常分支、超时与过期用例。
4. 运维：预览运行环境、域名代理、资源限制、监控告警。

## 6. 测试用例清单（MVP 必测）

1. `generate finished` 后自动触发预览并最终 `ready`。
2. `modify finished` 后自动触发预览并最终 `ready`。
3. 部署失败进入 `failed`，可重建恢复。
4. 过期后状态为 `expired`，可重建恢复。
5. 非目标任务类型不触发预览。
6. 下载 ZIP 与修改生成功能不受影响。
7. 接口鉴权生效，跨用户不可访问他人预览。

## 7. 发布与回滚清单

1. 发布前执行数据库迁移并校验索引生效。
2. 先在测试环境验证端到端流程后再灰度。
3. 灰度阶段开启 `preview.enabled=true`，观察成功率和耗时。
4. 异常时关闭 `preview.autoDeployOnFinish` 快速止损。
5. 保留下载链路作为回退兜底能力。
