# v2.1 页面体验与账户能力迭代 Phase 1~4 发布说明与回滚预案

## 一、发布内容概述
本次发布主要包含前端路由及信息架构的解耦、聊天等待态重构、异常重试机制以及后端可观测性增强。所有的功能均按计划完成，满足 PRD 要求。

### 1. 路由与页面解耦 (Phase 1)
- 新增 `/home` 路由及 `HomePage` 组件，作为默认登录成功后的导览页。
- 侧边栏独立拆分“首页”、“我的项目”、“对话”三个一级入口。
- 将原本耦合在 `ProjectListPage` 的主输入框移除，项目页仅保留项目列表管理能力。
- 在聊天页 `ChatPage` 支持 `sessionId=new` 状态，承接新的空态对话输入功能。

### 2. 聊天消息状态机与等待态 (Phase 2)
- 扩展了前端 `ChatMessage` 的 `status` 字段，支持 `pending`、`streaming`、`done`、`error` 状态跟踪。
- 引入了 `AssistantThinkingCard` 等待态组件，在 AI 响应的初期和生成期间提供清晰的视觉动效反馈。
- SSE 全链路无缝接入状态机，支持首包前显示 waiting 骨架屏，delta 时显示 streaming 呼吸灯。

### 3. 异常可恢复与后端可观测 (Phase 3)
- 前端完善了错误捕获机制，针对异常消息提供包含错误原因和【重试】按钮的红色提示卡片。
- 重试按钮会自动回退失败的回复，并重新发送最后一次的用户请求，不影响后续对话。
- 后端增加日志监控，支持计算并记录 `latencyMs` 与 `firstTokenLatencyMs`。
- 规范化超时和异常处理的映射，通过 SSE 下发带有 `code` 和 `message` 的 `error` 事件。

### 4. 视觉回归适配 (Phase 4)
- 对新引入的 `AssistantThinkingCard` 及 `streaming` 呼吸灯效果进行了暗色模式 (`dark:`) 类的全面适配，保证在深色主题下对比度舒适。

---

## 二、回滚点与预案
本次发版主要涉及前端 UI 及状态机的升级以及后端的日志打点优化，**未破坏现有数据表结构及核心 API 契约**。若出现严重阻断问题，可进行无缝回滚。

### 1. 前端回滚
- 若新的路由或等待态组件在特定环境下出现白屏或无限加载，可将前端静态资源直接回滚至上一版本 `v2.0`。
- 回滚后用户将继续使用耦合了新建对话功能的旧版 `ProjectListPage`，不会产生数据冲突。

### 2. 后端回滚
- 若新增的 SSE error 抛出机制或日志拦截逻辑导致服务资源泄漏，可将 `services/api-gateway-java` 回滚至 `v2.0` 对应的 commit。
- 本次后端修改仅涉及 `ChatService.java` 中 SSE 处理和耗时计算记录，无新增/删除数据表操作。回滚后端不会导致历史对话数据丢失。

---

## 三、验证与监控
1. 验证方法：
   - 登录系统，查看首页和侧边栏高亮是否符合预期。
   - 从首页/对话页发起新对话，观察响应阶段是否能在 100ms 内出现骨架屏与呼吸灯。
   - 关闭网络或强制超时，观察是否出现带错误码提示的重试按钮。
2. 观测指标：
   - 关注后端日志输出的 `Chat stream completed. Request latency: xxx ms, First token latency: xxx ms`，确认时延是否正常。
