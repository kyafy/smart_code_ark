# 智码方舟 PRD v1（MVP）

## 1. 文档信息
- 产品名称：智码方舟（thesis.polars.cc）
- 版本：v1.0（MVP）
- 阶段：Phase 0 产品边界与统一规范
- 目标用户：高校毕业设计学生（个人开发为主）

## 2. 产品目标
- 用户通过中文自然语言描述需求
- 系统通过对话梳理功能清单与数据库设计
- 用户选择技术栈后自动生成可运行项目工程
- 提供任务进度、下载交付包、预览与增量修改

## 3. MVP 范围（做什么/不做什么）
### 3.1 In Scope（MVP）
1. 账号体系：注册、登录、JWT鉴权
2. 项目管理：创建项目、项目列表、项目详情
3. 对话澄清：`/api/chat` 多轮会话
4. 结构化产出：功能清单 + 数据库表结构（SQL）
5. 一键生成：`/api/generate` 异步任务 + 状态查询
6. 交付下载：ZIP 下载（代码+SQL+API文档+部署脚本）
7. 预览能力：任务完成后返回预览 URL
8. 增量修改：`/api/task/{id}/modify` 中文指令更新
9. 计费积分：基础扣费、余额不足提示

### 3.2 Out of Scope（MVP不做）
1. 多租户企业版权限体系（组织/部门级）
2. 复杂工作流编排（BPMN级别）
3. 本地私有模型训练平台
4. App原生端（iOS/Android）专属生成器
5. 高级论文自动写作全流程（仅保留提纲辅助）

## 4. 首发技术栈白名单
- 前端：Vue3 + Vite + Pinia + ElementPlus
- 后端：Python FastAPI + Celery + Redis
- 数据库：MySQL 8.0
- 存储：OSS/S3
- 向量库（可选）：Milvus 或 Qdrant
- 模型：GPT-4/国产模型混合路由

## 5. 核心用户流程
1. 注册登录
2. 创建项目并输入需求
3. 对话澄清并确认功能清单
4. 选择技术栈并发起生成
5. 查看任务日志与进度
6. 下载项目包或打开预览
7. 输入修改指令并再次生成

## 6. 功能需求（MVP）
- FR-001 用户注册/登录/JWT鉴权
- FR-002 项目创建与配置（标题、项目类型、技术栈）
- FR-003 对话会话管理（历史消息、超时处理）
- FR-004 需求结构化（功能清单/表结构）
- FR-005 异步任务生成（状态机：queued/running/finished/failed）
- FR-006 任务日志推送（WebSocket/SSE）
- FR-007 交付包下载（ZIP）
- FR-008 预览地址获取
- FR-009 中文修改指令增量更新
- FR-010 计费扣费与流水记录

## 7. 非功能需求（MVP）
- 可用性：核心接口成功率 >= 99%
- 性能：`/api/chat` P95 < 3s（不含大模型长文本）
- 安全：全站HTTPS、密码哈希存储、JWT过期控制
- 可观测：任务耗时、失败率、Token成本可追踪
- 合规：展示“AI生成仅供参考”与学术诚信提示

## 8. 验收标准（退出标准）
- 团队确认首发范围与非范围清单
- API规范/Prompt规范/数据字典均评审通过
- 能指导开发直接进入 Phase 1（无关键阻塞）
