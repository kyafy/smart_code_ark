> 版本：v1.0

## 1. 规范目标

1. 明确三段式纠偏涉及的输入/输出契约。  
2. 明确一致性指标与写回判定规则。  
3. 明确错误语义、日志字段、审计字段与指标口径。  
4. 为后续实现提供“决策完成”的执行依据。  

---

## 2. 适用接口与契约边界

本规范仅覆盖以下类级契约：

1. `OutlineExpandStep`  
2. `ModelService.runPromptForJson`  
3. `QualityRewriteStep`  

不覆盖：

1. 新增 HTTP API 设计  
2. 前端字段协议变更  
3. RAG 算法策略替换  

---

## 3. OutlineExpandStep 契约

### 3.1 输入契约

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `flattenedOutline` | Outline 数据结构 | 是 | 已扁平化章节/小节结构 |
| `expectedChapterCount` | `int` | 是 | 目标章节总数，>0 |
| `expectedSectionCountPerChapter` | `Map<Integer, Integer>` | 是 | 章节索引到目标小节数映射 |
| `batchMetadata` | 批次元数据 | 否 | 分批扩写场景元信息 |

输入校验要求：

1. `expectedChapterCount <= 0` 视为无效输入。  
2. `expectedSectionCountPerChapter` 缺失或含非法值（<=0）视为无效输入。  

### 3.2 输出契约

| 字段 | 类型 | 说明 |
|---|---|---|
| `normalizedManuscript` | Manuscript JSON | 标准化后的正文结构 |
| `citationMap` | CitationMap JSON | 引用映射 |
| `chapterCoverage` | `double` | 章节覆盖率 |
| `sectionCoverage` | `double` | 小节覆盖率 |
| `preCommitDecision` | `enum` | 写回前判定 |

---

## 4. ModelService.runPromptForJson 契约

### 4.1 输入契约

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `template` | Prompt 模板 | 是 | 已解析模板 |
| `vars` | `Map<String, String>` | 是 | 渲染变量 |
| `retryContext` | 纠偏上下文 | 否 | 包含前次失败原因、重试计数等 |

### 4.2 输出契约

| 字段 | 类型 | 说明 |
|---|---|---|
| `jsonNode` | `JsonNode` | 模型解析结果 |
| `schemaValidationResult` | 校验结果对象 | schema 是否通过、错误详情 |
| `retryResult` | 重试结果对象 | 是否触发纠偏、重试次数、最终状态 |

### 4.3 纠偏控制要求

1. schema 违规时必须进入 corrective retry 流程。  
2. 超过重试上限时必须输出“重试耗尽”语义结果。  
3. 必须记录结构偏差分类原因。  

---

## 5. QualityRewriteStep 契约

### 5.1 输入契约

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `qualityIssues` | 质检问题集合 | 是 | 来自 `outline_quality_check` |
| `currentManuscript` | Manuscript JSON | 是 | 当前稳定正文 |
| `expectedChapterCount` | `int` | 是 | 目标章节数 |
| `expectedSectionCountPerChapter` | `Map<Integer, Integer>` | 是 | 目标小节数映射 |

### 5.2 输出契约

| 字段 | 类型 | 说明 |
|---|---|---|
| `rewrittenManuscript` | Manuscript JSON | 改写结果 |
| `preCommitValidationDecision` | `enum` | 写回前判定 |
| `structureRegression` | `boolean` | 是否发生结构回退 |
| `fallbackAction` | `enum` | 拒绝写回时回退动作 |

---

## 6. 结构一致性指标规范

### 6.1 指标定义

1. `chapterCoverage = generatedChapterCount / expectedChapterCount`  
2. `sectionCoverage = matchedSectionCount / expectedSectionCount`（全量 section 口径）  
3. `structureRegression = rewrite后结构 < rewrite前结构`（布尔）  

### 6.2 默认阈值策略

- 写回门槛（默认）：
  - `chapterCoverage = 1.0`
  - `sectionCoverage = 1.0`
  - `structureRegression = false`

- 不满足门槛时：
  - 禁止写回
  - 触发“重试或回退”分支

---

## 7. 写回契约（Pre-Commit Guard）

### 7.1 可写回判定

满足以下全部条件时允许写回：

1. `chapterCoverage` 达标。  
2. `sectionCoverage` 达标。  
3. 未发生 `structureRegression`。  
4. 关键正文内容未退化（空正文/占位文本被判为失败）。  

### 7.2 禁止写回判定

任一条件失败即禁止写回，并执行回退策略：

1. 优先保留上一稳定版本。  
2. 条件允许时触发纠偏重试。  
3. 仍失败时任务失败。  

---

## 8. 错误语义码（文档建议）

| 错误语义码 | 语义 |
|---|---|
| `PAPER_PRECHECK_INVALID_INPUT` | 生成前约束输入无效 |
| `PAPER_CORRECTIVE_RETRY_EXHAUSTED` | 生成中纠偏重试耗尽 |
| `PAPER_PRECOMMIT_COVERAGE_FAILED` | 写回前 coverage 未达标 |
| `PAPER_PRECOMMIT_REWRITE_REGRESSION` | 改写导致结构回退 |
| `PAPER_PRECOMMIT_WRITE_BLOCKED` | 写回被守卫阻断 |

说明：该语义码先用于文档契约，后续实现阶段映射到现有错误码体系。

## 9. 可观测性契约

### 9.1 必填日志字段

- `chapterCountBefore/After`

- 通过率（passRate）  
- 重试率（retryRate）  
- 写回阻断率（writeBlockRate）  
- 回退触发率（fallbackRate）

## 10. 测试矩阵与验收

### 10.1 单元测试（按三段式）

1. 生成前约束：目标节数注入、非法输入兜底。  
2. 生成中纠偏：schema 违规触发纠偏、重试耗尽。  
3. 写回前复检：coverage 不足阻断、rewrite 回退阻断。  

### 10.2 集成测试

1. `outline_expand -> quality_rewrite` 跨步骤结构一致性。  
2. 开关关闭时兼容旧链路。  

### 10.3 生产验收测试

1. 日志字段齐全且可检索。  
2. `prompt_history` 与落库结构一致。  
3. 回退动作可重放且可审计。  

### 10.4 DoD

1. 接口输入/输出/失败语义完整。  
2. 阈值、写回判定、回退路径明确。  
3. 不新增外部 API，不破坏现有协议。  

---

## 11. 默认假设

1. 本轮仅文档落地，不包含代码实现。  
2. 默认阈值采用严格一致策略（1.0/1.0）。  
3. 后续若需放宽阈值，必须通过配置化方案实现。
